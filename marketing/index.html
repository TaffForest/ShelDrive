SHELBY PROTOCOL

SHELDRIVE

Mount the decentralised web as a drive. Copy files in, they're pinned to the Shelby network. Open files, they're retrieved on demand. It's just a folder.

HOW IT WORKS
01
↑
WRITE
Copy or save files into the mounted drive. ShelDrive intercepts filesystem calls through FUSE and buffers content in memory.
cp file.txt /Volumes/ShelDrive/
02
◆
PIN
On file close, content is uploaded to the Shelby decentralised storage network. The returned CID is stored in a local SQLite index.
shelby:0x67d8.../file.txt
03
↓
READ
Open any file and ShelDrive retrieves it: first from memory, then disk cache, then the Shelby network. Transparent and instant.
cat /Volumes/ShelDrive/file.txt

APP SHELL
Tauri v2
Rust backend with native WebView. System tray integration, IPC commands, no Electron bloat. 8MB binary.
FILESYSTEM
fuser 0.17
Userspace filesystem driver. Intercepts OS read/write/delete calls and translates them to Shelby operations.
STORAGE INDEX
SQLite
Local path-to-CID mapping. Fast directory listings without network calls. Survives restarts. WAL mode.
FEATURES
■
Native Drive Mount
Appears in Finder, Explorer, and Nautilus as a real drive. No special tools needed to access your decentralised files.
FUSE / WINFSP
◇
Content-Addressed Index
Every file maps to a Shelby CID via a local SQLite database. Directory listings are instant. Lookups under 1ms for 10K+ files.
SQLITE
⚙
LRU Disk Cache
512MB local cache at ~/.sheldrive/cache/. Files survive app restarts. Automatic eviction when full. Read without network latency.
512 MB DEFAULT
↻
Self-Healing Sidecar
The Node.js SDK bridge auto-restarts up to 3 times on crash. Broken pipe detection with automatic reconnection.
AUTO-RESTART
⚠
Offline Resilience
Write files without network access. Content is saved to disk cache with a local CID and queued for upload when connectivity returns.
WRITE QUEUE
☷
Cross-Platform
macOS and Linux today via FUSE. Windows via WinFSP planned. Same codebase, same Tauri shell, platform-native installers.
