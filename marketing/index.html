How it works

Three steps. Completely transparent.

1

Write files

Copy or save files into the mounted drive. ShelDrive intercepts filesystem calls through FUSE and buffers content.

cp file.txt /Volumes/ShelDrive/
2

We pin it

On file close, content is uploaded to the Shelby decentralised storage network. The CID is stored in a local index.

shelby:0x67d8.../file.txt
3

Read on demand

Open any file and ShelDrive retrieves it from memory, disk cache, or the Shelby network. Transparent and instant.

cat /Volumes/ShelDrive/file.txt

Why ShelDrive?

Everything you need. Nothing you don't.

Native drive mount

Appears in Finder, Explorer, and Nautilus as a real drive. No special tools needed.

FUSE / WinFSP

Content-addressed index

Every file maps to a Shelby CID via SQLite. Directory listings are instant, lookups under 1ms.

SQLite + WAL

LRU disk cache

512MB local cache. Files survive restarts. Automatic eviction when full. Read without network latency.

512 MB

Self-healing sidecar

The Node.js SDK bridge auto-restarts up to 3 times on crash. Broken pipe detection with automatic reconnection.

Auto-restart

Offline resilience

Write files without network access. Content is cached locally and queued for upload when connectivity returns.

Write queue

Cross-platform

macOS and Linux today via FUSE. Windows via WinFSP planned. Same codebase, platform-native installers.

macOS / Linux / Windows

Tauri v2
Rust backend with native WebView. System tray integration, IPC commands, no Electron bloat.
FILESYSTEM
fuser 0.17
Userspace filesystem driver. Intercepts OS read/write/delete calls and translates them to Shelby operations.
STORAGE INDEX
SQLite
Local path-to-CID mapping. Fast directory listings without network calls. WAL mode, survives restarts.
